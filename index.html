    QDD Arm — Sim-to-Real Documentation¶

    The complete operators guide for the 6-DOF QDD arm sim-to-real stack: capture VLA datasets in sim and on real hardware, fine-tune GR00T N1.6 with domain randomisation, evaluate the policy in Isaac Lab and on the physical arm, and (if needed) co-train on a sim-real mix to close the gap.

    • Stage 1b — Real-Hardware Data Collection

      Same dataset format captured on the physical arm with two USB cameras (top + wrist; no sim involvement). Used for Stage 4 demos and the real-side of Stage 5 co-training.

    • Stages 2–5 — Sim-to-Real Training

      Domain-randomised dataset → GR00T N1.6 fine-tune → sim eval → real eval → co-training, with the full training command adapted to our 7-DoF arm.

    Stage cheat sheet¶

    Stage What runs Key knob Pass criterion
    1a — Sim capture scripts/launch_bridge.sh + scripts/launch_sim.sh agent_config.yaml → logging (rate, dataset_root) + dataset_config.yaml → task_ranges (prompts) + isaac_scene_config.yaml → domain_randomization (DR) 50–200 demos per task in addverb_sim_dataset/
    1b — Real capture addverb_arms/addverb_vla_datacollection/run_datacollect_docker.sh config_single.py 50–200 demos per task in addverb_single_arm_dataset/
    2 — Train gr00t/experiment/launch_finetune.py --base_model_path nvidia/GR00T-N1.6-3B, --max_steps 10000, --learning_rate 1e-4, --global_batch_size 32 Loss curve plateaus; checkpoint at 10 000 steps
    3 — Sim eval lerobot_eval against Addverb-Teleop-Cam-v0-Eval --action_horizon 16, DR on ≥ 50–70 % success over ≥ 30 rollouts
    4 — Real eval run_gr00t_server.py + real-arm client Same checkpoint as Stage 3 Acceptable real-world success rate
    5 — Co-train Re-run Stage 2 on sim+real mix 14 : 1 to 20 : 1 sim : real ratio (see the doc) Closes the sim → real gap

    Conventions¶

    • Joint vector is always 7 entries: [link_1..link_6] (rad) + slide_1 (m). slide_2 mirrors slide_1 automatically (env hook + USD axis flip). link_7 is welded as a PhysicsFixedJoint in the USD physics layer (the wrist flange is never actuated on hardware), so it has no DOF and never appears in the action / observation vectors.
    • Gripper kinematics is slider-crank, calibrated by a single reference angle physical_open_rad.
    • Single source of truth for launching is launch_config.yaml at the repo root. Both launch_bridge.sh and launch_sim.sh read it.
    • Dataset format is identical between sim and real captures (Parquet rows + per-episode JPEGs), so the two streams can be mixed for co-training (Stage 5).
    • Domain randomisation ranges live in isaac_scene_config.yaml → domain_randomization. Each block carries its own enabled: true|false flag (default false); only enabled blocks are wired as reset events at env build time.
    • Logging knobs (operational): rate, queue depth, debug-print frequency, output path → agent_config.yaml → logging. Dataset schema (image dims, jpeg quality, dof layout, task_ranges) → dataset_config.yaml. Task prompts come from dataset_config.yaml → task_ranges for sim captures, and from EPISODE_TASK_RANGES in addverb_vla_datacollection/config_single.py for hardware captures.
