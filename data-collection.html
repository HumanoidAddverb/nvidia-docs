    Data Collection — Addverb → Isaac Sim¶

    End-to-end recipe for capturing teleoperation episodes (hardware leader → Isaac Sim follower) into the canonical Parquet + JPEG dataset format used by addverb_vla_datacollection. Everything below assumes the repo root General_Autonomy_Special_Robots/.

    Above: hardware leader back-driven by hand, Isaac Sim follower mirroring in real time while SimDataLogger writes Parquet rows + JPEGs to disk.

    What gets recorded¶

    Per frame, written at agent_config.yaml → logging.fps (30 Hz by default), in addverb_vla_datacollection-compatible Parquet + JPEG. Logger queue depth (max_queue_depth) and debug-print frequency (debug_freq_hz) live in the same agent_config.yaml → logging block. Dataset SCHEMA (image dims, jpeg quality, dof layout, task prompts) is configured separately under dataset_config.yaml.

    Field Type Shape Meaning
    observation.state.joint_positions float32 (7,) Sim joint state — link_1..link_6, slide_1 (slide_2 mirrors slide_1, intentionally not logged)
    action.robot float32 (7,) Commanded joint targets (same order)
    state.base / action.base float32 (2,) Zeros (no mobile base)
    base.timestamp_us int64 (1,) Leader hardware timestamp (microseconds)
    observation.images.ego JPEG path Wrist-mounted RGB
    observation.images.external JPEG path Fixed isometric workspace RGB

    Image size and JPEG quality come from dataset_config.yaml → images (default 320×320, q=95).

    Prerequisites (one-time per machine)¶

    1. Docker image built (sim PC): 
      ./docker/build_sim.sh              # produces addverb-isaaclab:latest
    2. Conda env on host (hardware PC + sim PC alike): 
      conda env create -f environment.yml      # or pip-install addverb_isaaclab/requirements.txt
conda activate unified_addverb_isaaclab
    3. Robot USD bundle is at addverb_isaaclab/assets/usd/addverb_arm/addverb_gripper_urdf_2.usd (already in repo). scripts/launch_sim.sh aborts early if missing.
    4. CAN bus up (real-mode only): 
      ./addverb_arms/setup_all_cans.sh

    Calibration aligns hardware joint encoders + the gripper-actuator zero point with the sim. Run once after mounting the arm, and again any time the leader is dismantled or homing drifts.

    ./scripts/launch_calibration.sh
# overrides:
INTERFACE=can1 ./scripts/launch_calibration.sh
MODEL=ADD      ./scripts/launch_calibration.sh

    Interactive prompts:

    1. Move leader arm to the sim home pose (all 6 arm joints centred), press ENTER → averages 50 samples, writes joint_offset_rad.
    2. For each of the 6 arm joints (link_1..link_6), move it to its MAX positive extreme, press ENTER; then to its MAX negative extreme, press ENTER. 12 captures total; the script writes the resulting sim-frame range to joint_min_rad and joint_max_rad per joint. These FULLY OVERRIDE the URDF-mirroring defaults in agent_config.yaml — at sim-agent startup the mapper uses calibrated limits, never the USD-baked ones.
    3. Move the gripper to the fully-OPEN mechanical pose (the slider-crank zero), press ENTER → writes physical_open_rad.
    4. Type the gripper offset (in metres) — distance between slide pistons when the gripper is fully closed, measured with a calliper. Blank to skip. Writes gripper_offset_m.

    Output → addverb_isaaclab/config/calibration.yaml (the previous file is backed up to calibration.yaml.bak.<ts>). The launcher then drops into the normal publishing loop, so you can Ctrl+C and start a session, or just let it run and launch the sim in another terminal.

    KinematicMapper overlays calibration.yaml on top of agent_config.yaml at sim start. Watch the log for:

    [KinematicMapper] overlaid calibration from .../config/calibration.yaml

    1. Edit launch_config.yaml (one-line per session)¶

    mode: real           # 'mock' for hardware-free dry run, 'real' for the arm
sim:
  task:        Addverb-Teleop-Cam-v0
  leader_addr: tcp://localhost:5556          # cross-machine: tcp://<hw-ip>:5556
  repo_root:   /workspace/datasets/session_001    # path INSIDE the container
  episode_num: 0
bridge:
  conda_env:   unified_addverb_isaaclab
  config_path: addverb_isaaclab/config/bridge_config.yaml
  mock_sine:   true                          # mock-mode: send slow sine

    repo_root is interpreted inside the container — /workspace/datasets/ maps to <repo>/datasets/ on the host (mounted by docker/run_sim.sh).

    2. Start the bridge¶

    ./scripts/launch_bridge.sh                   # mode from launch_config.yaml
# Or force the mode:
MODE=real ./scripts/launch_bridge.sh
MODE=mock ./scripts/launch_bridge.sh

    The bridge stays attached to the terminal, prints joint frames at the configured control.loop_hz (200 Hz), and listens for keyboard control: - R — reset both physical arm (reset_to_home()) and the sim env - S — finalize the current dataset episode, home both arms, and start a fresh episode (auto-increments episode_num). Use this between takes — saves to disk before homing so the homing motion itself is never logged into the episode you keep. - E — clean shutdown of bridge + sim

    pynput is used in an X session; if no DISPLAY, an stdin-cbreak fallback takes over (terminal must have focus).

    3. Start the sim¶

    ./scripts/launch_sim.sh                      # uses launch_config.yaml as-is
EPISODE_NUM=3 ./scripts/launch_sim.sh        # bump episode without editing yaml

    The container brings up Isaac Sim with two floating viewport windows (WristCam, TableCam) showing the same RGB streams the logger writes. SimDataLogger starts in a background thread — the first prints look like:

    [SimDataLogger] config       : .../dataset_config.yaml
[SimDataLogger]   log image  : 320x320  jpeg_q=95
[SimDataLogger]   cameras    : ['ego', 'external']
[SimDataLogger]   joint/act  : 7/7
[SimDataLogger]   rate       : 30.0 Hz
[SimDataLogger] Episode 0 started → /workspace/datasets/session_001

    4. Move the leader, capture, end the episode¶

    • Back-drive the physical leader by hand. Sim follows in real time (effective end-to-end latency: ~50 ms, dominated by sensor render time).
    • The cube spawns randomly on the table at every reset. Pick + place it.
    • Press R on the bridge keyboard to reset for the next take.
    • Press E (or Ctrl+C) to exit cleanly. The logger flushes Parquet rows + writes meta/info.json before the env tears down.

    5. Multi-episode¶

    Four options, in order of preference:

    • Press S between takes (recommended) — finalizes the current episode to Parquet+JPEG, increments episode_num, starts a fresh episode, and homes both the physical leader and the sim arm. No Isaac Sim restart needed. Pair with R for an in-episode reset that does not save.

    • Restart with bumped episode_num — EPISODE_NUM=N ./scripts/launch_sim.sh. Use when you also want to reload yaml configs or pick up code changes; pays the Isaac Sim startup cost (~30 s).

    • Press R mid-session — resets the env (cube re-spawns); the current episode keeps growing under the same episode_num. Use for long-running rollouts in one episode.

    • Auto-home reset — if the operator returns the arm to home (all six arm joints within 0.03 rad of zero) for more than 50 frames, the agent triggers an env reset by itself. Useful for hands-off chained takes within the same episode.

    Output layout¶

    repo_root ends up as:

    <repo_root>/
├── meta/
│   ├── info.json                                 # codebase + robot + dof + image config
│   ├── tasks.parquet                             # task name + prompt per episode
│   └── episodes/chunk-000/file-000.parquet       # episode index (start row, length, ...)
├── data/chunk-000/file-000.parquet               # one row per logged frame
└── images/observation.images.<cam>/episode_<N>/<timestamp_us>.jpg

    info.json is filled in from dataset_config.yaml: 

    {
  "codebase_version": "v3.0",
  "robot_type":      "addverb_sim_6dof",
  "fps":             30.0,
  "joint_dim":       7,
  "action_dim":      7,
  "image_keys":      ["observation.images.ego", "observation.images.external"]
}

    tasks.parquet rows come from dataset_config.yaml → task_ranges. The default range covers all episode numbers [0, 9999] with prompt "Pick up the object and place it in the designated location.".

    Configuration reference¶

    File What lives here
    launch_config.yaml Single source of truth for launch_*.sh: mode, task, leader_addr, repo_root, episode
    addverb_isaaclab/config/bridge_config.yaml Physical arm: CAN interface, leader PD gains, ZMQ port, control-loop Hz
    addverb_isaaclab/config/agent_config.yaml Sim PC: ZMQ subscriber, KinematicMapper (sign flip / offset / gripper), logging
    addverb_isaaclab/config/isaac_scene_config.yaml Scene physics, robot mount pose, USD path, cameras, actuator gains, table, cube
    addverb_isaaclab/config/dataset_config.yaml Parquet schema (DOF dims), JPEG image size + quality, logging rate, task prompts
    addverb_isaaclab/config/calibration.yaml Auto-generated by --calibrate; overlays joint offsets + physical_open_rad on top of agent_config.yaml

    Important fields by file:

    # agent_config.yaml — KinematicMapper
kinematics:
  isaac_joint_names: [link_1_joint, ..., link_6_joint, slide_1_joint]   # 7 entries
  joint_sign_flip:   [1, -1, 1, 1, -1, 1, 1]                            # per-joint sign
  joint_offset_rad:  [0, 0, 0, 0, 0, 0, 0]                              # overlaid by calibration
  joint_min_rad:     [-1.57, ..., -1.57,  0.00]                         # arm rad / slide m
  joint_max_rad:     [ 1.57, ...,  1.57,  0.05]
  gripper:
    physical_open_rad: 0.0193     # actuator angle at fully-OPEN pose (theta=0 reference)
    gripper_offset_m:      0.0        # measured slide-piston gap when fully closed (overlaid by calibration.yaml)

# agent_config.yaml — SimDataLogger operational knobs
logging:
  enabled:         true                          # set true to actually write Parquet/JPEG

  debug_freq_hz:   1.0                           # console debug-print rate

    Gripper kinematics (slider-crank)¶

    Hardware actuator angle θ_raw → simulated slide position slide_m:

    theta    = θ_raw  -  physical_open_rad           # calibration zero
x_mm     = 35 · (1.6128 - cos θ - sqrt(1.249 - sin² θ))
x_clamp  = clamp(x_mm / 1000, 0, 0.05)
slide_m  = clamp((0.05 - x_clamp) + gripper_offset_m, 0, 0.05)

    The 0.05 - inverts the raw curve so the physical open pose maps to slide=0.05 (jaws OPEN at max travel) and the folded pose maps to slide=0 (jaws CLOSED). The trailing + gripper_offset_m bumps the fully-closed slider value up by the measured piston gap (so simulated jaws sit gripper_offset_m apart at full close instead of fused at zero); the outer clamp keeps the slide inside the URDF [0, 0.05] range. slide_m is written to both slide_1_joint and slide_2_joint each step. The USD authors slide_2's joint frame flipped 180° about X so equal positive joint values produce mirrored world motion (paddles separate). The ImplicitActuator PD gains (isaac_scene_config.yaml → actuators.gripper: stiffness 50000 N/m, damping 2000 N·s/m, effort 500 N) hold both stiffly.

    connect_1_2 and connect_2_2 are rigidly welded (PhysicsFixedJoint) to their respective slides — no closed-loop linkage to drive.

    Mock mode (no hardware)¶

    mode: mock                              # in launch_config.yaml
bridge.mock_sine: true                  # slow sine across all 7 joints

    ./scripts/launch_bridge.sh              # publishes synthetic joint frames
./scripts/launch_sim.sh                 # sim follows them

    Useful for end-to-end logger / pipeline checks without the leader arm.

    Cross-machine (bridge on a different PC)¶

    scripts/launch_sim.sh runs the container with --network=host so tcp://localhost:5556 always reaches the host's bridge. To put the bridge on a different machine, edit launch_config.yaml:

    sim:
  leader_addr: tcp://10.0.4.117:5556    # bridge PC's LAN IP

    The bridge config (bridge_config.yaml → zmq.host: 0.0.0.0) already listens on every interface, so no change needed on the bridge side.

    Troubleshooting¶

    Symptom Fix
    Robot USD not found in launcher Path mismatch — confirm assets/usd/addverb_arm/addverb_gripper_urdf_2.usd exists and isaac_scene_config.yaml → robot.usd_path matches
    [ZMQLeaderReceiver] leader link stale (Xs); holding pose Bridge died or stopped publishing — restart launch_bridge.sh; sim freezes the last command (no garbage written to dataset)
    Address already in use on bridge start Another bridge is bound to port 5556 — kill it or change bridge_config.yaml → zmq.port
    Sim joint names mismatch in agent log Update agent_config.yaml → kinematics.isaac_joint_names to match env.unwrapped.scene["robot"].data.joint_names
    Calibration overlay isn't applied Look for [KinematicMapper] overlaid calibration from ... in agent log; if missing, check calibration.yaml exists next to agent_config.yaml
    Sim slides don't open with hardware actuator (a) physical_open_rad wrong — re-calibrate; (b) hardware actuator angle isn't moving past the dead-zone — slider-crank gives x_m=0 for the first ~0.6 rad of motion (clamped negative); verify with KinematicMapper.describe() printout in the agent log
    Sim jaws move together instead of splitting apart USD slide_2_joint axis-flip lost — re-run tools/configure_gripper_joints.py to re-apply the 180° local-rotation flip
    pynput fails with DisplayNameError (real bridge, no DISPLAY) Launch the bridge inside an X session, or run with stdin focus and use the cbreak fallback
    High teleop latency Drop rendering.camera_update_period_s to ~30 ms or lower camera resolution in isaac_scene_config.yaml
    Cube falls through table table.center[2] + table.size[2]/2 must equal robot.init_pos[2]; cube's z is auto-computed from the table top — re-check both in isaac_scene_config.yaml
    Pressed S but no new episode started Check the agent log for saved episode N + started episode N+1. If only the saved line appeared, the next SimDataLogger constructor failed — usually a config typo in dataset_config.yaml. Logging stops; restart the agent to recover.
    Empty / corrupt Parquet at episode end Confirm the agent printed [SimDataLogger] Episode N started; if not, agent_config.yaml → logging.enabled is false or --repo_root isn't passed
